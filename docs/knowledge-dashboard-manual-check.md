# Knowledge Dashboard Manual Check

1. Backend lokal starten und `http://127.0.0.1:8001/dashboard` oeffnen.
2. Den sichtbaren Knowledge-Status pruefen.
3. Eine TXT-Datei hochladen und den Status `bereit` pruefen.
4. Eine PDF mit Text hochladen.
5. Eine XLSX-Datei mit mindestens zwei Arbeitsblaettern hochladen.
6. Eine CSV-Datei hochladen.
7. Mehrere harmlose Dateien gleichzeitig hochladen.
8. Eine gueltige und eine ungueltige Datei gemeinsam hochladen und den partiellen Erfolg pruefen.
9. Dieselbe Datei erneut hochladen und `Duplikat` pruefen.
10. Eine textlose PDF hochladen und `OCR erforderlich` pruefen.
11. Die Dokumentliste, Metadaten und begrenzte Details pruefen.
12. Die Knowledge-Suche ausfuehren und Fundstellen pruefen.
13. Ein Dokument neu indexieren und anschliessend eine normale Chatfrage dazu stellen.
14. Quellen nur unter der zugehoerigen Jarvis-Antwort pruefen; die Sprachausgabe darf nur die Antwort vorlesen.
15. Eine Antwort ohne passenden Dokumentkontext pruefen; sie darf keinen Quellenblock zeigen.
16. Ein Uploaddokument entfernen und die lokale Bereinigung pruefen.
17. Ein `local_path`-Dokument entfernen und pruefen, dass die Originaldatei erhalten bleibt.
18. Dashboard neu laden und die Persistenz der Liste pruefen.
19. Manuellen Voice-Button pruefen.
20. Desktop-Wake-WebSocket pruefen.
